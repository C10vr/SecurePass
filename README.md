Section A: Password analyzer
In this section the system will analyze the input password that the user had given. This system would check for any special symbols, lowercase letters, uppercase letters and numbers, with the length of the password would be taken to account. 'Check for data breaches' button will scour through the internet for any exposed data breaches that may contain the password the user had entered. (Still work in progress)

Section B: Password Generator
In this section the system will be able to provide the user a safer password rather than their normal password (e.g Ethan123, or qwerty123), This system would look into the users preference by selecting the length of the password as well as containing the other elements (Upper/Lowercase alphabets, special symbols and numbers). 

Section C: Threat Logger
In this section the system will look into the user's, the system will identify common threats from with the user's password. Rather than analyzing it to show if its weak or strong, the system will explain potential risk from these passwords and provide recommendation for improvement.

Section D: Attack Simulator
In this section the system will provide a simulator that will demonstrates how attackers attempt to crack weak passwords through different brute force techniques (Dictionary attacks, Brute forcing, Credential stuffing, Common password pattern matching, etc). The simulation would only be educational only and show estimated attack duration without performing any actual attacks.
